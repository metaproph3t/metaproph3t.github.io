Why the number of full nodes doesn't affect decentralization

In The Limits to Blockchain Scalability, Vitalik discusses why he thinks it's crucial for regular users to be able to run a node. Basically, his argument is that regular users verifying the chain is what keeps validators from introducing invalid state transitions. I think this logic is flawed for a few reasons described in this post.

Verification of state transitions doesn't matter unless it intermediates access

Imagine for a second that the recent whistleblower claims are true and that aliens really do exist on Earth. Now imagine that these aliens had, unbeknownst to us, been validating the Ethereum blockchain. Each block, they download all the block's transactions, apply them to their own copy of the state, and make sure that everything checks out. Are these aliens helping secure Ethereum against an attack like the one Vitalik describes?

Obviously, no, they are not. Although they are validating blocks, no (human) users are using this block validation to determine whether a chain is valid. A colluding group of validators could just as easily pull the wool over their eyes with an invalid block. In other words, verification of state transitions only matters insofar as it intermediates user access.

In reality, users are not intermediating access with full nodes

Ethernodes lists 6,720 distinct Ethereum nodes. According to this TechCrunch article, MetaMask has surpassed 30 million monthly active users (MAUs). This means that even if every single node operator was using their node to intermediate access to Ethereum, only 0.02% of users are accessing Ethereum via their own full node. The remaining 99.98% of users are trusting someone else to verify the chain on their behalf, like Infura, Etherscan, or Coinbase.

The geographic dispersion of nodes, visualized. The hotspot on the East Coast of the United States is Ashburn, VA, where Amazon has a data center.

In my view, this means that there is practically no benefit of lessening hardware requirements such that regular users can run nodes. The vast majority aren't going to validate anyway, so hardware requirements just need to be low enough that crypto service providers (exchanges, RPC providers, payment processors, etc.) can run nodes.

So what does matter for decentralization?

I don't want this post to get unwieldy, so I'll briefly list the things that do matter for decentralization:

  • Light / diet clients, which prevent a rogue RPC from tricking users
  • Fraud proofs, which prevent an attack like the one Vitalik described by giving an alarm system to light clients
  • Stake dispersion (a.k.a. having a high Nakamoto Coefficient), which makes attacks hard by increasing the threshold of nodes that would need to collude to attempt an attack
